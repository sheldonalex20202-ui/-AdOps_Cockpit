# Инфраструктура

## Полный стек на одну связку

Каждая рабочая связка требует следующей инфраструктуры:

| Компонент | Что именно | Примечания |
|---|---|---|
| **Ad Account** | Рекламный кабинет Facebook | Внутри BM |
| **BM** | Business Manager | Управляет кабинетами, пикселями, доменами, страницами |
| **Пиксель** | Facebook Pixel | Привязан к верифицированному домену |
| **Домен** | Верифицирован в BM | DNS TXT-запись через Cloudflare/DNS-провайдер |
| **Fan Page** | Facebook-страница | Привязана к BM и к рекламному кабинету |
| **Крео** | Рекламный материал | Несколько форматов и вариантов |
| **Лендинг** | Страница оффера | Белая + чёрная (с клоакой) или просто белая |
| **Трекер** | Keitaro / Binom | Между объявлением и лендингом |
| **Антидетект** | Dolphin Anty / Octo / Undetectable | Один профиль = один аккаунт |
| **Прокси** | Мобильный / резидентский | ГЕО прокси = ГЕО аккаунта |
| **Карта** | Виртуальная или физическая | Привязана к рекламному кабинету |

## Типы аккаунтов

| Тип | Описание | Использование |
|---|---|---|
| **Кинг (Мамка)** | Высокодоверенный, прогретый аккаунт | Распределяет траст. Рекомендуется иметь 2 кинга |
| **BM под кингом** | Business Manager, созданный с кинга | 5–7 BM с одного качественного аккаунта |
| **Фарм-аккаунт** | Вручную прогретый, 2–4 недели | Добавляется в BM под кингом |
| **Авторег** | Массово созданный аккаунт-расходник | К одному кингу — до 10 авторегов |
| **Купленный прогретый** | Покупается у фармеров | Сразу готов к работе |
| **Безлимитный BM** | Без ограничений на кол-во кабинетов | Покупается на специализированных площадках |

**Принцип распределения:** авторегов разделяют между двумя BM — при бане одного BM не потеряешь все аккаунты.

**Лимиты BM по расходам:**
- Новый BM: $50/день лимит
- BM с историей: $250/день
- Купленный "безлимитный BM": без ограничений

## Прогрев аккаунта (Farming)

### Фаза 1: Настройка окружения (День 0)

- Уникальный профиль в антидетект-браузере
- Выделенный резидентский или мобильный прокси (ГЕО = ГЕО аккаунта)
- Уникальный email-адрес (не переиспользовать)
- Уникальный номер телефона для SMS-верификации
- User Agent: iOS или Android (Windows-аккаунты флагаются сильнее в 2025)
- Куки с сайтов целевой страны (загружены заранее)

### Фаза 2: Наращивание профиля (Дни 1–7)

- Заполнить профиль: реальное имя, фото, биография, локация
- Симулировать органическое поведение: лента, лайки, группы, подписки на страницы
- Добавить 5–15 друзей постепенно
- Без рекламы и без создания BM
- Поддерживать паттерн входов: тот же прокси, тот же профиль, одинаковое время суток

### Фаза 3: Бизнес-настройка (Дни 7–14)

- Создать Fan Page
- Создать Business Manager
- Верифицировать домен
- Добавить и верифицировать платёжный инструмент
- Всё ещё без реальной рекламы

### Фаза 4: Прогрев рекламой (Дни 14–21+)

- Запустить первый адсет на белый оффер с бюджетом $1–5/день
- Постепенно увеличивать расход: не более 20–30%/день
- Только после стабильной работы — переключать на рабочую (чёрную) связку

### Факторы траста аккаунта

| Измерение | Детали |
|---|---|
| **Device Trust** | Стабильный fingerprint: canvas, WebGL, шрифты, разрешение, аудио |
| **IP Trust** | Резидентский/мобильный IP; не datacenter, не VPN; стабильное ГЕО |
| **Behavior Trust** | Органичные паттерны активности; нет массовых действий; паузы как у человека |
| **Account Age** | Чем старше аккаунт — тем выше базовый траст |

## Верификация домена (шаги)

1. Зарегистрировать чистый домен (желательно с историей или новый без "грязной" истории)
2. Проверить домен в Facebook по спискам бана (перед использованием)
3. BM → Brand Safety → Domains → Add → скопировать DNS TXT-запись
4. Добавить TXT-запись в Cloudflare / DNS-провайдер
5. Нажать "Verify" в Facebook
6. Привязать пиксель к верифицированному домену
7. Настроить AEM (Aggregated Event Measurement): приоритетные события (Purchase, Lead, ViewContent)

> Без верификации домена Meta не может эффективно оптимизировать конверсионные кампании (ограничение после iOS 14+).

## Пиксель: "Вечный пиксель" (стратегия)

- Каждый лендинг получает свой пиксель (или shared "мастер-пиксель")
- **"Донор-аккаунт"**: пиксель создаётся на аккаунте, который никогда не запускает рекламу (не будет забанен). Этот аккаунт делится доступом к пикселю с рабочими кабинетами.
- Если рекламный кабинет банят — пиксель (и вся накопленная аудитория) сохраняется у донора.
- CAPI (Conversions API) обязателен: без него теряется 20–40% событий из-за iOS и блокировщиков.

## Виртуальные карты

Каждый рекламный кабинет требует свою платёжную карту.

**Формат добавления карты в инструменты:**
```
1111222233335555;01;25;777
(номер; месяц; год; CVV)
```

**Популярные сервисы:**
- Combo Cards
- Capitalist
- Privacy.com (US)
- Специализированные сервисы для арбитражников

## Технические данные, нужные на запуск

### Минимум на один аккаунт

```
Facebook User Token        — извлекается из куки Ads Manager
Ad Account ID              — act_XXXXXXXXXX
BM ID                      — Business Portfolio ID
Facebook Page ID           — Fan Page
Pixel ID                   — 15-значный ID
CAPI Access Token          — из Events Manager → Data Sources → Settings
Domain                     — верифицирован в BM
Proxy credentials          — назначены этому аккаунту
```

### На каждую запускаемую связку

```
Offer URL                  — трекер-ссылка (Keitaro) с макросами
Creative files             — 1:1 и 9:16 для изображений; видео
Ad copy                    — headline, primary text, description
CTA type                   — SHOP_NOW / LEARN_MORE / SIGN_UP / GET_OFFER
UTM parameters             — utm_source, utm_medium, utm_campaign, utm_content
Target GEO                 — коды стран
Objective                  — OUTCOME_SALES / OUTCOME_LEADS / OUTCOME_TRAFFIC
Budget                     — daily или lifetime (в центах через API)
Bid strategy               — LOWEST_COST_WITHOUT_CAP / COST_CAP / BID_CAP
Start time                 — ISO 8601
Placement                  — Advantage+ или ручные позиции
Audience targeting         — интересы / lookalike / broad
```

### Для серых вертикалей (дополнительно)

```
Cloaking setup             — Keitaro flow, IM KLO, Cloak IT
White page URL             — для ботов/модераторов
Black page URL             — реальный оффер для пользователей
Pixel dedup event_id       — для дедупликации CAPI событий
Backup accounts            — прогретые запасные аккаунты
```
