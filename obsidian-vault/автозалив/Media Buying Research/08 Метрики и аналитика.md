# Метрики и аналитика

## Ключевые метрики

| Метрика | Описание | Ориентир |
|---|---|---|
| **CTR** | Click-Through Rate — кликабельность объявления | Выше 1–2% хорошо; для нутры/гемблинга |
| **CPC** | Cost Per Click — цена за клик | Зависит от ГЕО и вертикали |
| **CPM** | Cost Per Mille — цена за 1000 показов | Индикатор конкурентности аукциона |
| **CR / CVR** | Conversion Rate — конверсия клик → лид | Зависит от лендинга |
| **CPL** | Cost Per Lead — цена за лид | Должна быть ниже выплаты сети |
| **CPA** | Cost Per Action — цена за целевое действие | = CPL для лидогенерации |
| **Click2Reg** | Конверсия прелендинга (клик → регистрация) | Индикатор качества прелендинга |
| **ROI** | (Revenue − Spend) / Spend × 100% | >0% = в плюсе; >50% = отличный залив |
| **ROAS** | Revenue / Spend | >1 = прибыльно |
| **Frequency** | Среднее кол-во показов на одного пользователя | >2.5–3 = выгорание крео |
| **CR Checkout** | Клик → покупка (товарка) | Индикатор качества лендинга |

## UTM-структура для трекинга

Стандартная UTM-строка для Facebook:

```
utm_source={{site_source_name}}
&utm_medium=paidsocial
&utm_campaign={{campaign.name}}
&utm_content={{adset.name}}
&utm_term={{ad.name}}
```

Расширенный вариант (с FB-идентификаторами):

```
utm_source=facebook
&utm_medium=paid
&utm_campaign={{campaign.id}}
&utm_content={{adset.id}}
&utm_term={{ad.id}}
&creative={{ad.name}}
```

**Макросы Facebook:**
- `{{campaign.id}}` — ID кампании
- `{{campaign.name}}` — Название кампании
- `{{adset.id}}` — ID адсета
- `{{adset.name}}` — Название адсета
- `{{ad.id}}` — ID объявления
- `{{ad.name}}` — Название объявления
- `{{site_source_name}}` — Плейсмент (fb, ig, an...)
- `{{placement}}` — Конкретный плейсмент

## Пороги принятия решений

### Когда останавливать адсет

- CTR < 0.5% при Spend > $30
- CPL > 2× target CPA при Spend > 1.5× CPA
- 0 конверсий при Spend = 2× target CPA
- Frequency > 3.5 (выгорание аудитории)

### Когда масштабировать

- CPL стабильно ≤ target × 0.8 в течение 3+ дней
- ROAS > 3.0 при Spend > $50
- Стабильная фаза обучения завершена (5–7 оптимизационных событий)

### Когда увеличивать бюджет

- Не более +20–30% за один шаг
- Интервал: каждые 48–72 часа
- Резкий рост (2× и более) → сброс фазы обучения → не делать

## Фаза обучения Facebook

**Что это:** период, когда алгоритм FB оптимизирует доставку под аудиторию. Обычно 7–14 дней или 50 конверсий.

**Статусы:**
- `Learning` — в процессе обучения
- `Learning limited` — недостаточно данных (< 50 событий/неделю)
- (без пометки) — стабильный

**Что сбрасывает фазу обучения:**
- Изменение бюджета > 20–25% за раз
- Изменение аудитории, плейсмента, цели
- Переименование адсета
- Добавление/удаление объявлений

## Airtable / Notion — альтернативы Google Sheets

### Airtable Marketing Campaign Tracker

**Поля:** Campaign Name, Platform, Objective, Status, Start/End Date, Budget, Spend, Leads, Revenue, Assigned Buyer, Creative Assets (linked), GEO, Notes.

**Представления:**
- Grid (полные данные)
- Kanban (по статусу)
- Calendar (по дате запуска)
- Gallery (превью крео)

### Notion Media Asset Tracker

Linked databases: кампании → креативы → офферы.
Timeline view для планирования окон запуска.

## Экспорт/импорт из Ads Manager

### Экспорт для редактирования

1. Ads Manager → Export → Download as XLSX
2. Изменить: названия, URL, крео, UTM, бюджеты
3. Очистить ID-поля: `Ad Set ID`, `Ad ID`, `Campaign ID`, `Ad Set Time`
4. Импортировать обратно → новые кампании
