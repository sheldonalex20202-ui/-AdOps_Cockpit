# Нейминг (Naming Conventions)

## Зачем нужен нейминг

Единая система именования кампаний позволяет:
- Быстро ориентироваться в сотнях кампаний
- Фильтровать и анализировать данные в трекерах и BI-инструментах
- Автоматически парсить параметры в таблицах
- Передавать кампании между членами команды без объяснений

---

## Технические правила

1. Разделитель `_` (подчёркивание) между основными элементами
2. Дефис `-` внутри одного элемента (например, `ONLINE-SHOPPING`)
3. **Никогда**: пробелы, `&`, `?`, `/`, `#` — ломают SQL-запросы и URL-парсинг
4. Длина: до 50 символов (Ads Manager обрезает длинные в UI)
5. Дата: `YYYYMMDD` (сортируется) или `MMMYY` (читабельно): `Apr26`
6. **Никогда не переименовывать живые кампании** — сбрасывает фазу обучения и ломает связи в трекерах
7. Версии (V1, V2, V3) должны иметь задокументированный триггер
8. Все аббревиатуры — в едином "таксономическом документе" команды

## Нейминг по уровням

### Уровень кампании

**Универсальный шаблон:**
```
[Источник]_[Вертикаль]_[Оффер]_[ГЕО]_[Тип бюджета]_[Дата]
```

**Примеры:**
```
FB_GAMBL_CazinoX_BR_ABO_20260101
FB_NUTRA_SlimFit_PL_CBO_Apr26
FB_CRYPTO_BitcoinPro_DE_ABO_260115
```

**Agency-формат (с клиентом):**
```
[ClientCode]_[Objective]_[FunnelStage]_[Product]_[BudgetType]_[GEO]_[Date]
ACME_CONV_TOF_ProductLaunch_CBO_US_20260115
CLI01_FB_LEAD_TOFU_SummerSale_ABO_US-CA
```

**Compact-формат (малая команда):**
```
FB_CONV_LAL1%_AllPlacements_Mar2026
NUTRA_DE_SlimPro_CBO_260101
GMB_PL_CR1_ABO
```

### Уровень адсета

**Шаблон:**
```
[Тип аудитории]_[Аудитория/Интересы]_[Плейсмент]_[Событие]
```

**Примеры:**
```
BROAD-US-AllPlacements
LAL-1pct-Purchasers-FeedOnly
INT_Health_FB-Feed_Lead
RET-CartAbandon-EU-Stories
RETARGET_30d_IGS_Purch
1-Purch-US_LAL1%_Adv+_Install
DE_M25-45_Gambling_Broad
```

**Для залива на приложения (гемблинг):**
```
[GEO]_[Пол+Возраст]_[Тип аудитории]_[Плейсмент]
DE_M25-45_Broad_Adv+
PL_M18-35_LAL1pct_Feed
```

### Уровень объявления

**Шаблон:**
```
[Формат]_[Угол/Хук]_[Версия]
```

**Примеры:**
```
VID_WinStory_V1
IMG_Bonus100_red_V2
VID_Testimonial_Sarah_V1
IMG_BeforeAfter_V2
CAR_5cards_ProductLineup_V1
UGC_Hook1_9x16_V3
```

**Agency-формат:**
```
[ClientCode]_[Creative Name]_[AdType]_[Hook]_[Version]
ABC_SummerGlow_VID_Hook1_V2
ACME_SlimPro_IMG_DoctorAngle_V1
```

**Расширенный (с полной аналитикой):**
```
[Date]_[Product]_[Format]_[Creator]_[Concept]_[Angle]_[LP]_[Version]
20250815_SlimPro_VID_UGC_BeforeAfter_PainPoint_LP2_V1
```

## Нейминг для приложений (гемблинг / трекинг)

В параметрах трекинга deeplink / neyming строка кодирует все параметры:

```
sub1 = campaign_id         {{campaign.id}}
sub2 = offer_id
sub3 = PID (ID байера/вебмастера)
sub4 = landing_id
sub5 = creative_id
sub6–sub7 = собственные параметры
```

Пример полного макроса:
```
ankvwc_fehu_{{sub2}}_{{sub3}}_{{sub4}}_{{cr}}
fb_DE_betandreas_cr001_251015
```

## Z-группы (батчи тест-волн)

В CIS-командах принято нумеровать поколения тестируемых креативов:
- `Z1` — первая волна тестов
- `Z2` — вторая волна (обычно после выгорания Z1)
- `Z3` — и т.д.

Используется в нейминге адсета:
```
DE_M25-45_GMB_Z3       → 3-я волна крео, Германия, мужчины 25-45, гемблинг
PL_F35-55_NUTRA_Z1     → 1-я волна крео, Польша, женщины 35-55, нутра
```

## Нейминг по типу команды

### Соло-байер / малая команда (1–3 человека)

Простой формат, вертикаль-специфичные сокращения:
```
GMB_PL_CR1
NUT_DE_WL_V2
FB_CONV_RETARGET_VIDEO_V1
```

### Байер + ассистент

Включает инициалы байера:
```
IVN_GMB_PL_BetAndreas_CR2_260115
```

### Арбитражная команда (3–10 байеров)

Полный формат с идентификатором байера:
```
[Инициалы]_[Вертикаль]_[ГЕО]_[Оффер]_[Крео]_[Дата]
IVN_GMB_PL_BetAndreas_CR2_260115
ASD_NUT_DE_SlimPro_Z3_260201
```

### Крупное агентство (5+ байеров, несколько клиентов)

```
[ClientCode]_[Platform]_[Objective]_[GEO]_[Audience]_[Placement]_[Date]_[Version]
CLI01_FB_CONV_US_INT_FEED_2026-01-15_v1
ACME_IG_LEAD_DE_LAL1pct_STORIES_260201_v2
```

## Разделители: когда какой

| Разделитель | Использование |
|---|---|
| `_` (underscore) | Между основными элементами (уровнями нейминга) |
| `-` (hyphen) | Внутри элемента (составное значение): `ONLINE-SHOPPING`, `M25-45` |
| `\|` (pipe) | Некоторые агентства для читаемости в UI Ads Manager |

## Нейминг лендингов (рекомендация affLIFT)

```
[название-лендинга]_[версия]_[код-языка-страны]
```

**Примеры:**
```
norton-antivirus_black-header_$19-price_en-at
generic-loan_green_es-mx
slimpro_doctor-news_de
betandreas_win-scene_pl_v2
```

Правило: включать код языка, потому что один лендинг может переиспользоваться после перевода.

## Нейминг офферов в CPA-сетях

**Стандарт affLIFT / partnerkin:**

Минимально необходимо:
- Номер оффера в сети (например, `#12345`)
- ГЕО (`NG`, `PL`, `DE`)
- Название продукта (`WinPrize`, `BitcoinTrader`)
- Ограничения (carrier, device, flow)

**Примеры:**
```
#12345_NG_WinPrize_IVR_iphone
#98765_PL_BitcoinTrader_Mobile_SpringCarrier_$600
#11111_DE_OneLove_Dating_NoRestrictions
```
