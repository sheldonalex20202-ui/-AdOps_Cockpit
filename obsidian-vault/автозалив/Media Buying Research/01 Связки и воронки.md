# Связки и воронки

## Что такое связка

**Связка (bundle/funnel)** — это комбинация всех элементов, образующих воронку продаж. Рабочая формула:

> **Оффер + ГЕО + Крео = Связка**

### Полная структура связки

```
Источник трафика (Facebook Ads)
  → Креатив (крео): видео / изображение / карусель
  → [Клоака] → Белая страница (для модераторов FB)
             → Прелендинг (для реальных пользователей)
  → Лендинг (страница оффера)
  → Оффер (CPA-сеть → конверсия)
```

### Компоненты связки

| Компонент | Что это | Детали |
|---|---|---|
| **Оффер** | Партнёрский продукт | Из CPA-сети; имеет ID, выплату, ГЕО, ограничения |
| **ГЕО** | Целевая страна | Tier-1 (US/UK/AU), Tier-2 (Латам, Европа), Tier-3 (Азия, Африка) |
| **Крео** | Рекламный материал | Видео (.mp4), статика (.jpg/.png), карусель; минимум 3–5 вариантов |
| **Прелендинг** | Промежуточная страница | Прогревает пользователя, повышает Click2Reg; отдельный домен |
| **Лендинг** | Конечная страница оффера | С CTA, формой или кнопкой установки приложения |
| **Трекер** | Keitaro / Binom / Redtrack | Собирает аналитику: клики, лиды, CR по каждому элементу |
| **Клоака** | Разделение трафика | Модераторам → белая страница; реальным пользователям → деньги |
| **Пиксель** | Facebook Pixel + CAPI | Отслеживает события на лендинге, передаёт конверсии в FB |

## Пошаговый процесс залива

### Шаг 1 — Подготовка связки

1. Выбрать оффер в CPA-сети: вертикаль, выплата, ГЕО, условия
2. Согласовать условия с аффилиат-менеджером
3. Получить партнёрские ссылки / deeplink с трекинг-параметрами
4. Подобрать или создать крео:
   - Самостоятельно (видеограф / дизайнер)
   - Из спай-сервисов: AdHeart, Adplexity, Facebook Ad Library
5. Настроить трекер: добавить оффер, лендинг, прелендинг, прописать макросы
6. Протестировать воронку вручную: крео → прелендинг → лендинг → конверсия

### Шаг 2 — Подготовка аккаунта

1. Открыть антидетект-браузер, загрузить профиль аккаунта
2. Проверить прокси (ГЕО прокси = ГЕО аккаунта)
3. Зайти в FB, проверить доступ к Ads Manager и Business Manager
4. Проверить лимит рекламного кабинета (новые: $25–50/день)
5. Убедиться, что платёжная карта привязана и активна
6. Добавить Pixel или создать новый, настроить AEM

### Шаг 3 — Настройка кампании

**Уровень кампании:**
- Цель: App Installs / Conversions / Traffic (зависит от вертикали и оффера)
- Тип бюджета: ABO (тест) или CBO (масштаб)
- Нейминг по шаблону команды

**Уровень адсета:**
- ГЕО: целевая страна / регион
- Аудитория: возраст (обычно 25–55), пол, интересы
- Плейсмент: автоматический (Advantage+) или ручной (Feed, Stories, Reels)
- Бюджет: $30–50 на адсет для теста (цель: минимум 3–5 конверсий)
- Событие оптимизации: Lead, Purchase, Install, ViewContent
- Пиксель ID

**Уровень объявления:**
- Крео (видео или изображение)
- Ad copy: текст, хук в первых 3–6 строках
- URL: ссылка трекера (Keitaro) с UTM-параметрами
- Нейминг объявления по шаблону

### Шаг 4 — Тестирование

Базовые схемы тестирования (кампании : адсеты : объявления):

| Схема | Описание | Когда использовать |
|---|---|---|
| **1-1-1** | 1 кампания, 1 адсет, 1 объявление | Аккаунты с маленьким лимитом |
| **1-1-3** | 1 кампания, 1 адсет, 3 объявления | Тест 3 крео на одной аудитории (стартовый) |
| **1-3-1** | 1 кампания, 3 адсета, 1 объявление | Тест 1 крео на 3 аудиториях |
| **1-5-1** | 1 кампания, 5 адсетов, 1 объявление | Масштабный тест аудиторий |
| **3-3-1** | 3 кампании, 3 адсета, 1 объявление | Тест на разных аккаунтах / BM |

**Флоу:** ABO тест (3–7 дней) → найти победителя → перевести в CBO для масштаба.

### Шаг 5 — Мониторинг и оптимизация

1. Утренняя проверка: статусы кампаний, расход бюджета, CTR, CPC, CPL
2. Остановка неэффективных адсетов (CTR ниже порога, CPL выше целевого)
3. Масштабирование победителей:
   - **Вертикальное**: увеличение бюджета на 15–20% каждые 48–72 часа (резкий рост → сброс фазы обучения)
   - **Горизонтальное**: дублирование кампании на новые аккаунты или аудитории
4. Ротация крео при выгорании (Frequency > 2.5–3 → менять крео)
5. Добавление новых крео, тест новых углов

## Файлы и шаблоны перед заливом

| Элемент | Формат | Детали |
|---|---|---|
| Крео (видео) | .mp4 | Минимум 3–5 вариантов, разные хуки |
| Крео (статика) | .jpg / .png | Размеры: 1:1, 9:16, 4:5 |
| Прелендинг | HTML на хостинге | Отдельный домен, адаптирован под ГЕО/язык |
| Лендинг | URL оффера или собственный | Deeplink с трекинг-параметрами |
| Трекер-ссылка | URL с макросами | `{campaign_id}`, `{adset_id}`, `{ad_id}`, `{creative}` |
| Белая страница | HTML / URL | Для прохождения модерации |
| Нейминг | Шаблон команды | Документ с правилами именования |
| Таблица залива | Google Sheets | Список аккаунтов, бюджеты, статусы, метрики |

## CBO vs ABO

| Параметр | ABO | CBO |
|---|---|---|
| **Бюджет** | На каждый адсет вручную | Общий на кампанию, FB распределяет |
| **Тест** | ✅ Равное распределение | ❌ Алгоритм может убить слабые адсеты рано |
| **Масштаб** | Менее удобно | ✅ Алгоритм перекидывает бюджет в лучшие |
| **Минимум** | $30–50/адсет | $50–200+/кампанию |
| **Контроль** | Полный | Ограниченный |

## Типы офферов по вертикалям

| Вертикаль | Тип конверсии | Типичная выплата | Уровень сложности |
|---|---|---|---|
| **Нутра** | COD (Cash on Delivery) / SS | $15–35/лид | Средний |
| **Гемблинг** | FTD (First Time Deposit) | $30–200+/депозит | Высокий |
| **Крипто** | FTD / Lead | $400–1200/депозит | Высокий |
| **Дейтинг** | SOI / DOI / PPS | $1–30/регистрация | Средний |
| **Товарка** | Продажа / COD | % от чека | Низкий–средний |
| **Беттинг** | FTD / RevShare | $30–100+/игрок | Высокий |
| **Финансы** | Lead / FTD | $50–500/лид | Высокий |

## Связка: жизненный цикл

```
Поиск → Тест (ABO, малый бюджет) → Подтверждение (3–5 конверсий в плюс)
  → Масштаб (CBO, увеличение бюджета) → Горизонтальный масштаб (больше аккаунтов)
  → Выгорание (рост CPL, падение CTR) → Обновление крео или смена связки
```

Средний жизненный цикл рабочей связки: **2–8 недель** (зависит от вертикали, конкуренции, сезонности).
