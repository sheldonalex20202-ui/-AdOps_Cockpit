# Структура кампаний

## Три уровня иерархии

```
Ad Account
  └── Campaign           (цель, стратегия бюджета)
        └── Ad Set       (аудитория, бюджет, расписание, плейсмент)
              └── Ad     (креатив, текст, URL)
```

---

## Уровень кампании — ключевые поля

| Поле | Тип | Обязательно | Примечания |
|---|---|---|---|
| `name` | string | Да | Макс. 400 символов |
| `objective` | enum | Да | ODAX-значения (с API v16+) |
| `status` | enum | Да | `PAUSED` при создании — рекомендуется |
| `special_ad_categories` | array | Да | Пустой массив `[]` если нет; пропуск = ошибка 400 |
| `daily_budget` | int (центы) | Нет* | На уровне кампании = CBO |
| `lifetime_budget` | int (центы) | Нет* | Взаимоисключающий с daily_budget |
| `bid_strategy` | enum | Нет | LOWEST_COST_WITHOUT_CAP (по умолчанию) |

### Цели кампании (ODAX — обязательно с 2024+)

| API-значение | Использование в арбитраже |
|---|---|
| `OUTCOME_SALES` | E-commerce, партнёрка, конверсии на покупку |
| `OUTCOME_LEADS` | Лидогенерация, CPA-кампании |
| `OUTCOME_TRAFFIC` | Клики, просмотры лендинга |
| `OUTCOME_APP_PROMOTION` | Установки приложений (гемблинг, игры) |
| `OUTCOME_ENGAGEMENT` | Лайки, комментарии (прогрев страниц) |
| `OUTCOME_AWARENESS` | Охват (редко в арбитраже) |

> Устаревшие значения (`CONVERSIONS`, `LINK_CLICKS`) — не использовать с API v16+, вернут ошибку 400.

## Уровень адсета — ключевые поля

| Поле | Тип | Обязательно | Примечания |
|---|---|---|---|
| `name` | string | Да | |
| `campaign_id` | string | Да | ID родительской кампании |
| `billing_event` | enum | Да | Обычно `IMPRESSIONS` |
| `optimization_goal` | enum | Да | Зависит от цели кампании |
| `targeting` | object | Да | Объект targeting spec |
| `daily_budget` | int | Да* | Для ABO; в центах |
| `promoted_object` | object | Нет* | Для конверсий: pixel_id + event |
| `status` | enum | Нет | `PAUSED` / `ACTIVE` |
| `start_time` | datetime | Нет | ISO 8601 |

### Комбинации optimization_goal + billing_event

| Цель кампании | optimization_goal | billing_event |
|---|---|---|
| OUTCOME_SALES | `OFFSITE_CONVERSIONS` | `IMPRESSIONS` |
| OUTCOME_LEADS | `LEAD_GENERATION` или `OFFSITE_CONVERSIONS` | `IMPRESSIONS` |
| OUTCOME_TRAFFIC | `LINK_CLICKS` или `LANDING_PAGE_VIEWS` | `IMPRESSIONS` |
| OUTCOME_APP_PROMOTION | `APP_INSTALLS` | `IMPRESSIONS` |

### promoted_object (для конверсионных кампаний)

```json
{
  "pixel_id": "123456789",
  "custom_event_type": "PURCHASE"
}
```

## Таргетинг (targeting spec)

### Базовая структура

```json
{
  "geo_locations": {
    "countries": ["DE", "PL"],
    "regions": [{"key": "3847"}]
  },
  "age_min": 25,
  "age_max": 55,
  "genders": [1, 2],
  "publisher_platforms": ["facebook", "instagram"],
  "facebook_positions": ["feed", "story", "reels"],
  "device_platforms": ["mobile", "desktop"]
}
```

### Типы аудиторий

| Тип | Описание | Код в нейминге |
|---|---|---|
| Broad (широкая) | Только ГЕО + возраст, без интересов | `BROAD` |
| Advantage+ Audience | FB сам ищет аудиторию | `ADV+` |
| Interest-based | По интересам/поведению | `INT` |
| Lookalike 1% | Похожая аудитория на 1% | `LAL1pct` |
| Lookalike 2–5% | Масштаб с LAL | `LAL2pct`, `LAL5pct` |
| Custom Audience | Загруженная база клиентов | `CUSTOM` |
| Retargeting | Посетители сайта / видевшие рекламу | `RET` или `RETARGET` |

### Advantage+ Audience (2025 рекомендация Meta)

```json
{
  "geo_locations": {"countries": ["US"]},
  "targeting_automation": {"advantage_audience": 1}
}
```
Meta сам определяет всё остальное. Рекомендован как default-подход в 2025.

> **Важно:** С марта 2025 детальные таргетинг-исключения по интересам удалены из Ads Manager и API.

## CBO vs ABO: полное сравнение

| Параметр | ABO (Ad Set Budget Optimization) | CBO (Campaign Budget Optimization) |
|---|---|---|
| **Бюджет** | Задаётся вручную на каждый адсет | Один бюджет на кампанию; FB распределяет |
| **Тест** | ✅ Равномерный расход — честный A/B | ❌ Алгоритм может убить слабые адсеты рано |
| **Масштаб** | Ручное увеличение каждого адсета | ✅ FB автоматически перекидывает в лучшие |
| **Мин. бюджет** | $30–50/адсет (3–5 конверсий) | $50–200+/кампанию |
| **Контроль** | Полный | Ограниченный |
| **Фаза обучения** | Отдельная на каждый адсет | Одна на кампанию |

**Рекомендованный флоу:**
```
ABO тест (3–7 дней) → найти работающую связку → CBO масштаб
```

## Форматы креативов (Ad)

### Изображение (Image Ad)

```json
{
  "object_story_spec": {
    "page_id": "PAGE_ID",
    "link_data": {
      "image_hash": "IMAGE_HASH",
      "link": "https://tracker.url/",
      "message": "Primary text (макс. 125 символов)",
      "name": "Headline (макс. 40 символов)",
      "description": "Description (макс. 30 символов)",
      "call_to_action": {"type": "LEARN_MORE"}
    }
  }
}
```

### Видео (Video Ad)

```json
{
  "object_story_spec": {
    "page_id": "PAGE_ID",
    "video_data": {
      "video_id": "VIDEO_ID",
      "image_url": "https://thumbnail.url/",
      "message": "Primary text",
      "title": "Headline",
      "call_to_action": {"type": "LEARN_MORE"}
    }
  }
}
```

### Карусель (Carousel Ad)

- Минимум 2, максимум 10 карточек
- Каждая карточка: изображение/видео + заголовок + ссылка

### Dynamic Creative (A/B сразу в одном объявлении)

```json
{
  "asset_feed_spec": {
    "images": [{"hash": "HASH1"}, {"hash": "HASH2"}],
    "bodies": [{"text": "Текст 1"}, {"text": "Текст 2"}],
    "titles": [{"text": "Заголовок A"}, {"text": "Заголовок B"}],
    "call_to_action_types": ["LEARN_MORE", "SHOP_NOW"],
    "link_urls": [{"website_url": "https://tracker.url/"}]
  }
}
```
Meta автоматически собирает и тестирует комбинации. Макс: 10 изображений + 10 видео.

## Bulk-создание кампаний

### Метод 1: Graph API Batch Requests (нативный)

- До **50 операций в одном батче**
- Считается как **1 API-вызов** для rate limiting
- Поддерживает зависимые операции через `{result=INDEX:$.id}`

### Метод 2: Async Batch Requests

- Для 100+ объектов
- Возвращает `handle`, опрашиваешь до статуса `COMPLETED`

### Метод 3: Python SDK / цикл по аккаунтам

Наиболее популярен у команд с автозаливом:
```python
for account in account_list:
    campaign = AdAccount(account['id']).create_campaign(
        params={
            'name': build_name(account, offer, geo),
            'objective': 'OUTCOME_SALES',
            'status': 'PAUSED',
            'special_ad_categories': [],
            'daily_budget': account['budget'],
        }
    )
```

### Метод 4: Ads Manager XLSX Import

1. Экспортировать существующую кампанию как XLSX
2. Дублировать строки для новых вариантов
3. Очистить: `Ad Set ID`, `Ad ID`, `Campaign ID`, `Ad Set Time`
4. Изменить: названия, URL, креативы, UTM, бюджеты
5. Импортировать обратно → создаются новые кампании

## Авто-правила (Automated Rules)

Срабатывают каждые **30 минут** против всех объектов.

### Типичные правила в арбитраже

**Kill rule (автоотключение убыточных):**
- Условие: Spend > 1.5× target CPA И Conversions = 0 (последние 3 дня)
- Действие: Отключить адсет

**Scale winner (масштаб победителей):**
- Условие: ROAS > 3.0 И Spend > $50
- Действие: Увеличить дневной бюджет на 20%

**Fatigue alert (выгорание крео):**
- Условие: Frequency > 3.5 (последние 7 дней)
- Действие: Отправить уведомление

**Budget cap (ограничение расхода):**
- Условие: Spend > $500 (lifetime)
- Действие: Отключить адсет

> Лимит: 250 активных правил на рекламный кабинет.

## Rate Limits

| Тип ограничения | Значение |
|---|---|
| API-вызовы | ~200 score-units/час на app per user |
| Размер батча | 50 операций |
| Рекламных материалов | ~5,000 на кабинет |
| Объявлений в адсете | 50 (рекомендованный макс.) |

## Breaking Changes 2024–2025

- **Устаревшие цели** (с v16+): обязательно использовать `OUTCOME_*`
- **Детальные таргетинг-исключения** удалены (март 2025)
- **Консолидация интересов** (июнь 2025): многие конкретные интересы объединены в широкие категории
- **Advantage+ Audience** — рекомендованный по умолчанию подход
- **CAPI** — фактически обязателен; pixel-only теряет 20–40% событий (iOS 14+)
- **ASC/AAC API** устарел: унифицированная структура Advantage+ в v25.0 (Q1 2026)
